Problem

Cheap bluetooth earbuds reconnect at max volume because they use hardware/absolute volume.

Normally:

Earbuds store volume internally
↓
Reconnect
↓
Earbuds tell Linux: "volume = 100%"
↓
PipeWire obeys
↓
ears explode

Fix

Disable Bluetooth hardware volume sync so Linux controls volume instead.

Create config:

~/.config/wireplumber/wireplumber.conf.d/bluez-monitor.conf

monitor.bluez.properties = {
    bluez5.enable-hw-volume = false
}

Restart audio:

systemctl --user restart wireplumber pipewire pipewire-pulse

Reconnect Bluetooth.
